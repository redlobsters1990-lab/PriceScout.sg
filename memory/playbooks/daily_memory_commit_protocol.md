# Daily Memory Commit Protocol

At the end of today's execution:

1. Review today's run memory.

2. Identify information that should be promoted to long-term knowledge.

Promote only if:
- a niche opportunity was validated
- a rejection reason is reusable
- a SERP pattern is repeatable
- a monetization source was confirmed
- a dataset structure was identified
- a page blueprint proved useful

3. Write promoted items to:
- `./memory/knowledge/`

4. Write today's operational summary to:
- `./memory/daily_state/{YYYY-MM-DD}.md`

Include:
- opportunities discovered
- opportunities rejected
- opportunities approved
- patterns learned
- unresolved tasks
- carryover items

5. Move raw evidence files to:
- `./memory/archive/{YYYY-MM-DD}/`

6. Keep current run outputs in:
- `./memory/runs/{run_id}/`

Do not duplicate data across folders.

7. Produce a memory log:
- `./memory/runs/{run_id}/memory_commit_log.md`
